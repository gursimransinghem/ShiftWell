    **Section 1: Algorithm Overview**
    Name the algorithm (e.g., "ShiftWell Circadian Stress Index" or "SCSI"), its lineage (based on chosen model from comparison doc), and its purpose: given a 14-day shift schedule, produce a stress score and pre-adaptation recommendation for each identified transition.

    **Section 2: Inputs**
    Define the exact input object:
    ```typescript
    interface PredictionInput {
      shifts: Array<{
        date: string;        // yyyy-MM-dd
        startHour: number;   // 0-23
        endHour: number;
        type: 'day' | 'evening' | 'night' | 'off';
      }>;
      currentSleepDebt: number;    // hours, from debt engine
      baselineMidsleep: number;    // Current circadian anchor point (hours, 0-24)
      lookAheadDays: number;       // 14 for Phase 22
    }
    ```

    **Section 3: Algorithm Steps**
    Numbered pseudocode:
    1. Scan shifts for transitions (consecutive days where startHour shifts by > 4h)
    2. Classify each transition using existing TransitionType enum
    3. For each transition, calculate circadian phase shift required (delta hours)
    4. Apply fatigue model formula to compute predicted alertness nadir
    5. Map nadir to severity: critical (<40%), high (40-55%), medium (55-70%), low (>70%)
    6. Calculate adaptation window: days before transition needed for pre-adaptation protocol
    7. Generate TransitionPrediction object for each identified transition

    **Section 4: Outputs**
    ```typescript
    interface TransitionPrediction {
      transitionDate: string;         // Date of the stressful transition
      transitionType: TransitionType;
      severityScore: number;          // 0-100 (higher = more stressful)
      severity: 'low' | 'medium' | 'high' | 'critical';
      preAdaptationStartDate: string; // When to begin pre-adaptation (3-5 days before)
      protocolType: string;           // Which Phase 9 protocol to trigger
      predictedAlertnesNadir: number; // Estimated alertness % at transition nadir
      daysUntilTransition: number;
    }
    ```

    **Section 5: Thresholds and Calibration**
    Document all magic numbers with scientific justification:
    - 4h shift threshold for transition detection (Eastman & Burgess 2009)
    - 3-5 day pre-adaptation window (Crowley et al. 2003)
    - Severity bands (derived from SAFTE validation data, Hursh et al. 2004)

    **Section 6: Edge Cases**
    Define behavior for:
    - No transitions in 14-day window (return empty array, no alerts)
    - Consecutive transitions with no recovery gap
    - Unknown shift types
    - Calendar gaps (days with no shift data)
    - Sleep debt > 8h entering a transition (escalate severity one tier)

    **Section 7: Integration Notes for Phase 22**
    - Which existing files to extend vs create new
    - Performance target: full 14-day scan < 50ms on device
    - How TransitionPrediction maps to Phase 9 protocol triggers

    Update `docs/research/SLEEP-SCIENCE-DATABASE.md` with new citations from both tasks under "Predictive Scheduling & Fatigue Models".
