    **Step 2: Create feedback-engine.ts**

    File header:
    ```typescript
    /**
     * Algorithm Feedback Engine — HealthKit Closed Loop
     *
     * Adjusts planned sleep windows based on systematic deviation between
     * planned and actual sleep timing (from HealthKit data).
     *
     * Algorithm: Exponential Moving Average (EMA) of bedtime deviations.
     * Scientific basis: Process S homeostatic calibration (Borbely 1982),
     * EMA smoothing for noisy actigraphy (Phillips et al. 2017).
     *
     * HK-11 — HRV-calibrated dead zone:
     * When overnight HRV is below the user's 20th percentile, the dead zone
     * expands from 20 to 30 min. Low-HRV nights produce noisier sleep timing
     * data (Phillips et al. 2017), so we suppress adjustments on those nights
     * to prevent the algorithm from adapting to compromised recovery readings.
     */
    ```

    Constants:
    ```typescript
    const SMOOTHING_ALPHA = 0.3;
    const DEAD_ZONE_MINUTES = 20;              // normal dead zone
    const HRV_LOW_DEAD_ZONE_MINUTES = 30;      // HK-11: expanded on low-HRV nights
    const HRV_LOW_PERCENTILE_THRESHOLD = 20;   // below 20th percentile = low night
    const MAX_DELTA_MINUTES = 30;
    const MIN_NIGHTS_REQUIRED = 3;
    const MAX_GAP_NIGHTS = 3;
    const CONVERGENCE_TARGET = 15;
    const STALL_THRESHOLD = 30;
    ```

    Two exported functions:

    1. `computeRecoveryScore(comparison: SleepComparison, hrvPercentile?: number): number`
       - Formula: `0.4 × timing_accuracy + 0.3 × duration_accuracy + 0.3 × hrv_component`
       - `timing_accuracy = clamp(1 - Math.abs(comparison.bedtimeDeviationMinutes) / 60, 0, 1) * 100`
       - `duration_accuracy = clamp(1 - Math.abs(comparison.durationDeviationMinutes) / 60, 0, 1) * 100`
       - `hrv_component = hrvPercentile ?? 50` (neutral when not available)
       - Returns 0-100 rounded to 1 decimal

    2. `computeFeedbackOffset(discrepancyHistory, protocol, previousOffset?, hrvContext?): FeedbackResult`
       - Guard 1: `validNights.length < MIN_NIGHTS_REQUIRED` → inactive
       - Guard 2: `protocol && protocol.transitionType !== 'none'` → inactive, convergenceStatus='converging'
       - Guard 3: consecutive missing nights >= MAX_GAP_NIGHTS → inactive
       - HK-11 dead zone selection: `const deadZone = (hrvContext && hrvContext.currentRMSSD < hrvContext.p20RMSSD) ? HRV_LOW_DEAD_ZONE_MINUTES : DEAD_ZONE_MINUTES`
       - EMA on recentNights.slice(-7), compute smoothedBedtime and smoothedWake
       - Apply dead zone: if `Math.abs(smoothedBedtime) < deadZone` → bedtimeDelta = 0, else `clamp(smoothedBedtime * 0.5, -MAX_DELTA_MINUTES, MAX_DELTA_MINUTES)`
       - Same for wake delta
       - Convergence status: converged if avgAbsDeviation <= CONVERGENCE_TARGET, stalled if 14+ nights and > STALL_THRESHOLD
       - Return FeedbackResult with activeDeadZoneMinutes set to whichever dead zone was used
