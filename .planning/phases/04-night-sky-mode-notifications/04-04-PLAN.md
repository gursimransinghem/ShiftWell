**Create src/hooks/useNightSkyMode.ts:**

The hook detects wind-down window, computes fillFraction, and returns overlay props. Key rules:
- Read `plan` from `usePlanStore((s) => s.plan)` — NO new setInterval
- Read `windDownLeadMinutes` from `useNotificationStore((s) => s.windDownLeadMinutes)`
- Filter for Night Sky eligible block: `b.type === 'main-sleep' && b.priority === 1` AND block starts between 18:00 and 12:00 next day (hour >= 18 || hour < 12) — prevents nap false-positives (Pitfall 4)
- `minutesUntilSleep = differenceInMinutes(nextSleep.start, now)`
- `isActive = minutesUntilSleep <= windDownLeadMinutes && minutesUntilSleep >= -nextSleepDurationMinutes` (still active during sleep window)
- `fillFraction` formula:
  - Base: `clamp(plannedSleepMinutes / (profile.sleepNeed * 60), 0, 1)` where profile comes from `useUserStore((s) => s.profile)` and plannedSleepMinutes = differenceInMinutes(nextSleep.end, nextSleep.start)
  - If now > nextSleep.start (user is past bedtime): apply penalty `fillFraction -= (minutesPastBedtime / plannedSleepMinutes) * 0.5`
  - clamp result to [0, 1]
- `tomorrowSchedule`: find blocks where start is after nextSleep.end AND start is before `addDays(startOfDay(nextSleep.end), 1)`, sorted by start, take first 3, map to `.label`
- Return: `{ isActive, minutesUntilSleep, fillFraction, alarmTime: nextSleep?.start ?? null, latestWakeTime: nextSleep?.end ?? null, tomorrowSchedule }`
- Wrap in `useMemo` with deps `[plan, windDownLeadMinutes, profile]`

Use imports: `useMemo` from react; `differenceInMinutes, isAfter, addDays, startOfDay` from date-fns; `usePlanStore` from @/src/store/plan-store; `useNotificationStore` from @/src/store/notification-store; `useUserStore` from @/src/store/user-store.
