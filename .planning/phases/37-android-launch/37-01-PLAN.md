Note: `google-services.json` requires a Firebase project (for push notifications on Android). Document in `user_setup` — Sim must create Firebase project and download this file.

Note: Health Connect permissions are declared in the app manifest, but actual permission request happens at runtime via the Health Connect API.

**src/lib/healthkit/android-health.ts — Google Health Connect adapter:**

Create an Android-compatible adapter with the same interface as hrv-reader.ts so score-store works unchanged on both platforms.

```typescript
import { Platform } from 'react-native';
// Note: react-native-health (used for iOS) does NOT support Android
// Android requires: @kingstinct/react-native-healthkit alternative or Health Connect SDK
// Use: react-native-health-connect (community library for Health Connect API)
// Install: npx expo install react-native-health-connect

export async function checkHealthConnectAvailability(): Promise<boolean>
// Health Connect requires Android 9+ (API 28) and Google Health Connect app installed
// Returns false on Android < 9 or iOS — graceful fallback

export async function requestHealthConnectPermissions(): Promise<boolean>
// Requests read permissions for:
// HealthConnectRecordType.SleepSession
// HealthConnectRecordType.HeartRateVariabilitySdnn  
// Returns true if both granted

export async function fetchOvernightHRV(
  sleepWindowStart: Date,
  sleepWindowEnd: Date
): Promise<number | null>
// Mirrors iOS interface from hrv-reader.ts — score-store calls this same function on Android
// Query HeartRateVariabilitySdnn records in sleep window
// Return mean value or null

export async function fetchSleepData(
  date: string  // ISO date "YYYY-MM-DD"
): Promise<{ startTime: string; endTime: string; durationMinutes: number } | null>
// Query SleepSession records for specified night
// Return sleep session details or null
```
