    **Step 2: Create `src/lib/circadian/prediction-engine.ts`**

    Implement `scanUpcomingTransitions(input: PredictionInput): TransitionPrediction[]`:

    1. Iterate shifts pairwise (day[i], day[i+1]) to find startHour delta > 4h
    2. For each detected transition, call `classifyShiftTransition(fromType, toType)` — reuse existing classify-shifts logic
    3. Compute circadian phase shift: `Math.abs(shift[i+1].startHour - shift[i].startHour)` hours
    4. Apply simplified SAFTE-derived alertness model (per PREDICTION-ALGORITHM-SPEC.md Section 3):
       - Base alertness at transition = 100 - (phaseShiftHours * 8) - (currentSleepDebt * 4)
       - Clamp to 0-100 range
    5. Map alertness to severity bands (per spec Section 5)
    6. Apply sleep debt escalation (debt > 8h → escalate one tier)
    7. Calculate preAdaptationStartDate = transitionDate minus adaptationDays:
       - critical: 5 days before, high: 4 days before, medium: 3 days before, low: no pre-adaptation
    8. Map severity to protocolType string (reuse Phase 9 protocol type names)
    9. Return sorted array (soonest transition first)

    Implement `buildPreAdaptationProtocol(prediction: TransitionPrediction, today: Date): PreAdaptationStep[]`:
    - Returns array of daily steps from preAdaptationStartDate to transitionDate
    - Each step: `{ date: string; action: string; shiftMinutes: number }` (shift bedtime N minutes earlier/later each day)
    - Distribute total phase shift evenly across adaptation days (max 90 min/day per Eastman & Burgess 2009)
