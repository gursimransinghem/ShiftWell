    **Step 1: Create `app/(onboarding)/calendar.tsx`** (D-01, D-02, D-03, D-05)

    This screen has three sequential phases, managed by local state `phase: 'connect' | 'calendars' | 'review'`:

    **Phase 1 — Connect (initial view):**
    - Header: "Connect Your Calendar" (heading2 style, TEXT.primary)
    - Subtext: "ShiftWell reads your shifts and writes your sleep plan back" (body style, TEXT.secondary)
    - Two CalendarProviderCard components stacked vertically:
      - Apple Calendar card: onConnect calls `requestCalendarAccess()`. On success, calls `fetchAppleCalendars()`, then `getOrCreateShiftWellCalendar()`, then `useCalendarStore.getState().connectApple(calendars, shiftWellCalId)`. Transitions to phase 'calendars'.
      - Google Calendar card: onConnect calls `GoogleSignin.signIn()` then `GoogleSignin.getTokens()`. With accessToken, calls `fetchGoogleCalendarList(accessToken)`. Stores via `connectGoogle(calendars, accessToken, Date.now() + 3600000)`. Transitions to phase 'calendars'. Import GoogleSignin from `@react-native-google-signin/google-signin`.
    - Bottom: "Skip for now" pressable text (TEXT.secondary, underlined). Navigates to next onboarding screen (same route as completion).

    **Phase 2 — Calendar Selection:**
    - Header: "Your Calendars" (heading2)
    - Subtext: "All calendars are included by default. Toggle off any you want to exclude." (body, TEXT.secondary)
    - CalendarToggleList component with all connected calendars (combine apple + google from store)
    - "Continue" button (ACCENT.primary) at bottom → transitions to phase 'review'

    **Phase 3 — Shift Review:**
    - Fetch events: call `fetchAppleEvents` and/or `fetchGoogleEvents` for enabled calendars, next 28 days (use `addDays(new Date(), 28)` from date-fns)
    - Run `shiftConfidence()` on each event. Store as `{ ...event, confidence }` array.
    - Render ShiftReviewList with the scored events
    - Local state tracks which events user confirmed as shifts (pre-populated based on confidence thresholds)
    - "Confirm Shifts" button at bottom → saves confirmed shifts to shifts-store (using existing addShift or batch method), then navigates forward

    **Navigation:**
    After confirm (or skip), call `router.push('/(tabs)')` — onboarding complete, go to main app. (The exact next route depends on whether this is the last onboarding step; read healthkit.tsx for the current terminal navigation pattern and match it.)

    **Error handling:**
    - If Apple permission denied: show inline message "Calendar access denied. You can enable it in Settings." with a link to `Linking.openSettings()`.
    - If Google sign-in fails: show inline message "Google sign-in failed. Try again or skip for now."
    - Wrap all async calls in try/catch. Show error state inline, never crash.
