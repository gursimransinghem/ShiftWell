Produce HIPAA-COMPLIANCE-ASSESSMENT.md mapping all data flows in the employer platform and specifying required safeguards.

**Section 1 — Does HIPAA Apply?**
ShiftWell operates as a consumer app (users download directly), NOT as a covered entity's business associate. However, when an employer purchases ShiftWell:
- Employer IS a covered entity (hospital, health system)
- ShiftWell becomes a Business Associate (BA) under HIPAA
- BAA (Business Associate Agreement) is required before any employer data sharing
Conclusion: BAA required for enterprise contracts. Define what this means for development.

**Section 2 — Data Flow Map**
Draw out each data flow and classify it:
1. User → Mobile App: sleep schedules, adherence data, recovery scores (PHI if linked to identifiable employee)
2. Mobile App → ShiftWell Backend: same data, encrypted in transit (TLS 1.2+)
3. ShiftWell Backend → Employer Dashboard API: AGGREGATED only, de-identified per HIPAA Safe Harbor
4. Employer → ShiftWell (schedule import): shift schedules with employee IDs — this IS PHI, requires BAA
5. ShiftWell Backend → Analytics: de-identified aggregate data

**Section 3 — De-identification Methods**
Two HIPAA-approved methods:
- Safe Harbor (45 CFR §164.514(b)): Remove 18 identifiers. List all 18 and which ShiftWell collects.
- Expert Determination: Statistical expert certifies residual risk is "very small." Required for cohorts < 50.
Recommendation: Safe Harbor for aggregate metrics. Expert determination + differential privacy for small cohorts.

**Section 4 — Differential Privacy for Small Cohorts**
For employer cohorts with < 50 users, pure de-identification may allow re-identification. Specify:
- Epsilon value: ε = 1.0 (standard for healthcare analytics)
- Mechanism: Laplace noise added to aggregate statistics
- Threshold: Apply differential privacy when cohort < 50 users
- Implementation: Server-side before API response (not in mobile app)

**Section 5 — Required Technical Safeguards (HIPAA §164.312)**
- Encryption at rest: AES-256 for all PHI in database
- Encryption in transit: TLS 1.3 minimum
- Access controls: Role-based access, employer sees only their cohort
- Audit logging: All PHI access logged with timestamp, user, action
- Minimum necessary: Employer dashboard receives only pre-aggregated metrics, never individual records

**Section 6 — Required Administrative Safeguards**
- BAA template: Draft terms (Covered Entity obligations, ShiftWell BA obligations, breach notification 60-day window)
- Data retention: PHI retained maximum 7 years per HIPAA, deleted on contract termination
- Breach response: Incident response plan outline
- SOC 2 Type II roadmap: Timeline and cost estimate for eventual certification

**Section 7 — Development Checklist for Phase 27**
Checkbox list of what must be true before first employer pilot:
- [ ] BAA template reviewed by healthcare attorney
- [ ] All PII stripped before aggregate query results leave backend
- [ ] Differential privacy applied for cohorts < 50
- [ ] TLS 1.3 on all API endpoints
- [ ] Access control: employer JWT scoped to their org_id only
- [ ] Audit log table created and populated on all PHI reads

**Section 1 — Market Overview**
Enterprise wellness market: $61B (2023), growing to $94B by 2030. Healthcare sector is largest vertical (28%). Key buyers: hospital HR departments, large health systems, self-insured employers. Budget: $200-500/employee/year for comprehensive wellness programs.

| Platform | Focus | Sleep Features | Shift Worker? | Price |

**Section 3 — Critical Gaps in Existing Solutions**
1. No platform offers circadian-aware scheduling — they all treat sleep as "hygiene" not science
2. No platform integrates with shift schedule to generate personalized sleep windows
3. No platform uses the Two-Process Model or any validated circadian algorithm
4. No platform offers pre-adaptation protocols before schedule transitions
5. No platform connects shift patterns to measurable recovery outcomes
6. ROI calculators in existing platforms use generic wellness data, not sleep-specific outcomes

**Section 4 — ShiftWell Competitive Positioning**
ShiftWell Enterprise is the only platform that:
- Generates individualized circadian sleep windows from actual shift schedules
- Measures adherence at the individual level and aggregates to cohort
- Quantifies the circadian cost of specific schedule patterns (employer-facing insight)
- Provides science-backed ROI tied to sleep quality, not generic wellness engagement

Positioning: "Workforce sleep science" vs "employee wellness program." Target buyer: CNO/VP Nursing + HR (not just HR alone).

**Section 5 — Pricing Recommendation**
Based on competitive landscape and value delivered:
- Tier 1 (Standard): $5/seat/month — cohort dashboard + basic metrics

- Tier 3 (Enterprise): $15/seat/month + BAA — full API access + manager alerts + custom integrations
- Pilot pricing: $0 for 90 days (3-month proof-of-concept, 50+ employees)

Volume discounts: 500+ seats: 20% off, 2000+ seats: 35% off.
