  <action>
    Create `.planning/phases/24-intelligence-validation-sprint/DATA-ANALYSIS-PIPELINE.md`:

    **Section 1: Data Sources**
    Define exactly what data to extract for the 90-day analysis:
    - AI Brief data: `ai-store.briefs[]` — generatedAtISO, adherencePercent, tokensUsed, passedGuardrails
    - Weekly brief engagement: open rate proxy (time between generation and first app open after Monday 8AM)
    - Predictive scheduling data: `prediction-store.predictions[]` — severityScore, transitionDate, whether pre-adaptation was followed
    - Pre-adaptation follow-through: plan adjustments made in the 3-5 day window before high/critical transitions vs control (no adjustments made)
    - Pattern alerts: `pattern-store.alerts[]` — type, severity, dismissed (dismissal = engagement), time to dismiss
    - Outcome data: adherencePercent (weekly trend), sleepDebtHours (weekly trend), discrepancy mean (weekly trend)

    For each data source: field name, store location, how to compute if derived.

    **Section 2: Cohort Definition**
    Minimum inclusion criteria for analysis:
    - 90+ days since app install
    - At least one weekly brief received (AI feature engaged)
    - Minimum 30 nights of HealthKit or plan adherence data
    - Active shift schedule (not all-off days)

    Exclusion criteria: users who disabled all AI features, users with < 14 days of data.

    Target: 50+ users meeting inclusion criteria.

    **Section 3: Primary Outcome Measures**
    Per RES-12 requirements:
    1. Adherence rate change: pre-AI (weeks 1-4) vs post-AI (weeks 9-12) — paired t-test
    2. Sleep debt trajectory: linear regression slope over 90 days (negative = improving)
    3. Mean discrepancy reduction: week 1-2 baseline vs week 11-12 endpoint
    4. Weekly brief engagement rate: briefs opened / briefs generated

    **Section 4: Secondary Outcome Measures**
    1. Predictive scheduling efficacy: transition recovery time (days until discrepancy returns to baseline) with pre-adaptation vs without
    2. Pattern alert actionability: adherence improvement in the 2 weeks following a pattern alert vs control weeks
    3. AI recommendation follow-through: correlation between brief engagement and next-week adherence change
    4. Guardrail pass rate: what % of generated briefs pass all guardrails (target >99%)

    **Section 5: Statistical Methods**
    - Paired t-test for pre/post adherence (parametric if N>30 per central limit theorem)
    - Wilcoxon signed-rank as fallback if distribution is non-normal (Shapiro-Wilk test first)
    - Linear regression for trajectory analysis
    - Pearson/Spearman correlation for engagement-outcome relationship
    - Effect size: Cohen's d for continuous outcomes (target d > 0.3 for clinical relevance)
    - Significance threshold: p < 0.05 (two-tailed)

    **Section 6: Analysis Script Pseudocode**
    Step-by-step procedure for running the analysis when data is available:
    1. Export raw data from AsyncStorage using developer tools
    2. Load into analysis environment (Python/pandas or R)
    3. Apply inclusion/exclusion criteria
    4. Compute primary and secondary measures
    5. Run statistical tests
    6. Generate tables and charts for report

    **Section 7: Reporting Template**
    Column headers and format for each required table in the 90-day report.
